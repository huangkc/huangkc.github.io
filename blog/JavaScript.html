10/2/2015

The last few posts introduced Ruby objects such as arrays and hashes, enumerable module, and classes. Numbers, strings, and booleans are the main data types in both Ruby and JavaScript and objects let us group these and other data types to create many interesting and practical applications.

To access values inside this object we can use the dot notation. For example, person.firstName would return the value "John" and person.age would return the value 32. To add an "address" property to "person," we can write:

person.address="123 Main Street".

Notice that in the examples we are creating a single person object using the literal notation var variableName={ }. The function can take any number of arguments and we can then define specific persons in a way similar to defining a variable. To create a person with the same attributes as the person in the hash example, we would write:

var john = new person("John", "Doe", 32).

Note the use of the keyword this instead of a variable name. It acts as a placeholder for the new object. When a new person object is created using the constructor, the value of this becomes the new object. Last thing to point out is that whether created using the literal notation or the constructors function, JavaScript objects may take functions as their properties. This can opens up many opportunities for designing interactions among objects, which allows us to design, among other things, interactive web pages.
