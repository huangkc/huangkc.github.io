Ruby Classes

9/24/2015

We know that objects are the primary programming constructs in Ruby and that objects have methods. In this post we will distill these concpets further by examining classes. In the code "ruby".length, ruby is a string object and #length is a method. When we say "ruby" is a string object, we mean that "ruby" is an object (or instance) of the string class. Every object in Ruby belongs to a class, and class provides object with methods. The number 1 is an object in the integer class, and mathematical operators such as +, -, /, and * are methods that are applicable to integers.

We can think of classes as a way to organize objects with similar attributes and methods. Once the bleprint is drawn up, we can build as many objects as we want and give different attribute to each object without having to redraw the blueprint every time.
