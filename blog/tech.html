Recursive Methods in Ruby

10/10/2015

This post introduces recursion, a fundamental concept in programming. When we have a programming problem, we learned to break down the problem into smaller pieces. With problems that involve multiple repeated operations, iteration is a good option. Recursion is another approach, one that involves having a method or function call itself.

We can illustrate recursive methods in Ruby using the "finding the factorial problem." If we were to write a method to find the factorial of 3, we could break 3*2*1 into (3*2)*1. If n=3, then n-1=2 and n-2=1. So we are looking for n*(n-1)*(n-2). Using iteration, we would assign a variable the value 1 and multiply it by n, n-1, n-2, and so on until n reaches 1 (since we are only interested in finding factorials for positive numbers here). This would result in the method evaluating the product of the arguments that are passed in each time. When 1 is passed in at the end, recursion stops. Recursion is about break down problems as well as division of labor. The two main parts of a recursive method are: the end goal, or base case, and the process of reducing the task towards the end goal. In the factorial example, the end goal is the condition n=1 and the process of reducing the task is to call the method itself and pass in a smaller n.

Recursive methods have many applications, for example in economic forecasting and financial modeling. But they can take up memory in the computer if the procesdure is very complex and sometimes this leads to stack overflow. As is with any method, it is important to weigh the advantages and disadvantages and determine if the situation is appropirate before adopting it.
